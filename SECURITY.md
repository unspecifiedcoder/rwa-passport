# Security Policy

## Reporting Vulnerabilities

Please report security vulnerabilities privately via email: **security@xythum.io**

Do **not** open public issues for security vulnerabilities.

## Scope

- Smart contracts in `src/`
- CCIP message handling and replay protection
- ZK proof verification and nullifier tracking
- Signature validation and signer registry
- Access control and authorization logic

## Out of Scope

- Frontend UI vulnerabilities
- Social engineering
- Issues in third-party dependencies (OpenZeppelin, Chainlink, Uniswap) — report these upstream

## Response Timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 7 days
- **Patch timeline**: within 30 days for critical issues

## Bug Bounty

A formal bug bounty program will be announced prior to mainnet launch. Responsible disclosures made before that will be retroactively eligible.
